# 派送計畫 — 釜山行程「年輕人版」研究 v2

> 建立時間：2026-04-25
> 對象：25-35 歲（含心輕的 38 歲 😎）朋友團
> 目的：重新探索 IG / 小紅書當紅景點 + 美食 + 活動，**淘汰老氣 / 觀光客陷阱**，產生「年輕人精選」候選清單給主 Claude 評估
> 與 v1 的差別：v1 偏全面（什麼都查），v2 偏品味（只挖年輕當紅）

## 拆分原則
- 每個迷你 agent 負責 **1 個主題、1 個地理區或人群面向**
- 每個 agent 搜尋上限 **8 次** WebSearch
- 嚴格逐項 flush（每寫完 1 項立即覆寫整檔）
- **資料來源優先序**：Instagram > 小紅書 > TikTok > Threads > YouTube vlog > Reddit > Google Maps reviews
- **不再用 PTT**（上一代年輕人，25-35 不滑了）
- 戰術：Google 搜「{關鍵字} site:instagram.com」「{關鍵字} 小紅書」「{關鍵字} TikTok」命中第三方整理；不直接 WebFetch IG/TikTok 個別貼文（連 2 次失敗就停手）
- 報告必附 IG / 小紅書 / TikTok 搜尋連結讓使用者自驗

## 預期完成時間
5-10 分鐘（每個 agent 範圍小、搜尋預算緊，按釜山 v1 實測迷你 agent 平均 3-5 分鐘）。

## 失敗回復策略
若某個 agent timeout：
1. 檢查報告檔是否有部分內容（H2 + 段落）
2. 把失敗 agent 拆成更小範圍重派（每次 1-2 項）
3. 三次都失敗 → 在報告裡標「⚠️ 此項無資料」並繼續

## 完成後評估流程

主 Claude 收齊 7 份報告後：
1. 拿新 v2 報告 + 原 v1 11 份對照
2. **產出「年輕人精選」最終建議清單**：
   - 每個 v1 站列「保留 / 砍掉 / 替換成 X」
   - 新發現的店家加進 Day 1-6 對應位置
3. 給使用者選擇：「全套套用 / 部分套用 / 我自己挑」
4. 確認後重排 final-itinerary.md
