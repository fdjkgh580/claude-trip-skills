# /trip-research — 深度研究目的地

使用與使用者相同的語言回覆（預設繁體中文）。根據使用者的旅行者畫像和目的地，派出研究 agent 進行深度調查。

## 前置檢查

1. 讀取當前目錄的 `CLAUDE.md`，確認目的地、日期、旅行風格等資訊
2. 讀取 `./traveler-profile.md`，取得旅行者畫像
3. 如果任一檔案不存在，提醒使用者先執行 `/trip-plan`

## 第一階段：前置風險調查（必做、先於主體研究）

在派出主體研究 agent 之前，**先派一個簡短的前置調查 agent**，目的是提早發現「簽證來不及」「護照不符合」這類會讓規劃白做的致命問題。

### 派出前置 agent

Prompt 要求：
- 查目的地國家對台灣護照持有人的簽證政策（免簽 / 落地簽 / 電子簽證 / 需事先申請）
- 簽證辦理所需工作天數（含郵寄 / 預約面試的時間）
- 護照效期要求（通常要求入境時剩餘 6 個月以上）
- 當地是否強制旅遊保險（申根、部分國家要求）
- 每個結論都必須附資料來源 URL（外交部領事局、各國官方網站）

報告存為 `.claude/reports/agent-0-gate.md`。

### 讀取結果後的紅黃綠燈判斷

從 CLAUDE.md 取出發日期，算「距出發天數」。從畫像取「護照狀態」。

護照辦理 / 換發時程（台灣）：一般件 10 個工作天、急件 3 個工作天。

| 條件 | 判定 | 行動 |
|------|------|------|
| 距出發天數 > 簽證所需時間 × 2（且護照沒問題） | 🟢 充裕 | 直接繼續主體研究，在總結中順帶提醒辦理時程 |
| 距出發天數 > 簽證所需時間，但不到兩倍；或護照需換發但時程勉強夠 | 🟡 來得及但要立刻辦 | 在繼續主體研究前，明確告訴使用者：「這幾件事請**今天或明天**就開始辦」。清單化列出 |
| 距出發天數 < 簽證所需時間（或護照已過期且不夠急件時程） | 🔴 來不及 | **停下來**，用 `AskUserQuestion` 問使用者要怎麼辦：改期 / 改目的地 / 取消；選了再決定要不要繼續研究 |

🔴 時的 `AskUserQuestion`：

| 選項 | 說明 |
|------|------|
| 改期，我可以晚點出發 | 請使用者給新日期，重新計算後再判斷 |
| 改目的地（免簽或容易辦的） | 回到 /trip-plan 重選目的地 |
| 取消這次規劃 | 結束流程 |
| 我知道風險，堅持繼續 | 使用者自負責任，繼續但在每份報告開頭標紅字警告 |

## 研究 Agent 策略

根據畫像中的**研究深度**決定 agent 數量：

### 標準方案（2 個平行研究）

將研究主題合併為 2 個 agent：

| Agent | 合併主題 |
|-------|---------|
| A | 核心體驗路線 + 飲食攻略 + 當季特殊活動 |
| B | 機票搜尋（如需要）+ 交通方式 + 安全實用資訊 + 住宿推薦 |

### 深度方案（最多 5 個平行研究）

| Agent | 主題 |
|-------|------|
| R1 | 核心體驗路線 |
| R2 | 飲食攻略 |
| R3 | 機票搜尋（如需要）+ 交通方式 |
| R4 | 安全與實用資訊 |
| R5 | 住宿推薦 |

完成後接續 R6（當季特殊活動），讀取 R1-R5 結果後再跑，品質更好。
如果使用者已有機票，R3 只做交通方式。

## 各 Agent 研究內容指引

以下是每個研究主題的內容。根據旅行者畫像動態調整研究重點。

### 核心體驗路線（根據畫像的「核心興趣」調整）

| 使用者興趣 | 研究重點 |
|-----------|---------|
| 攝影 | 光線最佳時段、非觀光客拍攝機位、日出日落時間、器材建議 |
| 美食 | 在地人餐廳、市場、需預約的名店、當地特色菜、省錢吃法 |
| 歷史文化 | 博物館精選路線、遺跡、免費導覽、避開人潮的時段 |
| 戶外冒險 | 步道難度分級、裝備需求、天氣影響、安全注意事項 |
| 建築藝術 | 建築風格路線、免費參觀時段、最佳觀賞角度 |
| 放鬆度假 | 海灘/溫泉/spa 比較、預約方式、淡旺季價差 |
| 購物 | 在地品牌、市集時間、退稅流程、避免觀光客陷阱 |
| 夜生活 | 酒吧/Live House 推薦、安全區域、當地飲酒文化 |
| 混合型 | 根據使用者提到的多個興趣綜合研究 |

### 飲食攻略（根據畫像的「飲食」和「預算」調整）

- 每日飲食花費試算（早/午/晚）
- 超市/便利商店省錢策略
- 當地必吃食物清單（附 Google Maps 連結格式：`https://www.google.com/maps/search/{店名+街道或地址}`，搜尋字串一律帶上街道名或完整地址，避免同名店家誤搜）
- 飲食限制者的替代方案（素食、過敏、清真）
- 飲水資訊（自來水能不能喝、免費飲水點）

### 機票搜尋（如果使用者尚未訂機票）

如果 `CLAUDE.md` 中沒有航班資訊，且環境中有 Kiwi.com MCP（`mcp__kiwi-com__search-flight`），則：
- 用 MCP 搜尋符合日期和出發地的航班
- 呈現 3-5 個選項，附比較表（價格/轉機次數/飛行時間/航空公司）
- 標注最便宜、最快、最佳 CP 值
- 提醒使用者機票價格浮動，建議盡早訂

如果沒有 Kiwi.com MCP，用 WebSearch 搜尋航班資訊，提供參考價格和建議訂票平台（Skyscanner、Google Flights 等）。

如果使用者已有機票，跳過此步驟。

### 交通方式

- 機場到市區的所有選項（價格/時間比較）
- 市內交通（地鐵、公車、計程車 App、步行路線）
- 交通票券比較（日票、多日票、單程票哪個划算）
- 城際交通（如果多城市：火車/巴士/租車比較、訂票教學）
- 根據旅伴調整：家庭帶小孩 → 計程車/包車多；背包客 → 大眾運輸為主
- 根據體力調整：行動不便 → 無障礙路線資訊
- 根據旅伴調整（家庭帶幼兒）：地鐵/車站電梯分佈、推車可搭乘的路線、主要景點無障礙入口位置

### 安全與實用資訊

- 目的地整體治安評估
- 常見詐騙手法（具體到地點和手法）
- 扒竊高風險區域和時段
- 緊急聯絡資訊（急診醫院、報案方式、台灣駐外館處）
- 保險理賠須知（報案證明取得流程）
- 插座規格、電壓
- 小費文化
- 廁所資訊（付費/免費）
- 根據旅伴調整：獨旅 → 單獨行動安全；女性獨旅 → 額外注意事項；家庭 → 兒童安全
- 根據旅行經驗：第一次出國 → 詳細說明；老手 → 精簡版

### 住宿推薦

- 推薦 3-5 間，附比較表（價格/評分/位置/特色）
- 根據住宿偏好篩選類型
- 根據預算篩選價位
- 位置建議（靠近哪個區域最方便）
- Check-in/out 時間、行李寄放、WiFi 等實用資訊
- Google Maps 連結
- 家庭旅行加查：家庭房/連通房可行性、廚房設備（嬰幼兒需要加熱食品）、洗衣機（小孩換洗量大）、嬰兒床是否需另外申請

### 當季特殊活動

- 旅行期間的節日、慶典、市集
- 季節限定體驗（楓葉、櫻花、極光、雪季等）
- 特殊展覽、音樂節、體育賽事
- 國定假日對商店營業時間的影響（非常重要！）
- 根據核心興趣篩選相關活動

## Agent Prompt 撰寫指引

派出 agent 時，每個 agent 的 prompt 必須包含：
1. 目的地和旅行日期
2. 從畫像中提取的相關偏好（不要把整個畫像丟進去，只取相關部分）
3. 明確的研究範圍和產出格式
4. 要求附上 Google Maps 連結（格式：`https://www.google.com/maps/search/{名稱+街道或地址}`，搜尋字串一律帶街道名或完整地址）
5. 要求以使用者的語言撰寫報告（從對話語言判斷）
6. 使用 `subagent_type: "general-purpose"` 確保有網路搜尋能力
7. 要求 agent 若搜尋不到可靠資訊，在報告中標記「⚠️ 資料不足，建議出發前自行確認」，不得捏造資訊
8. **Source citation 強制要求**：每個關鍵事實（票價、營業時間、餐廳地址、交通時刻等）都必須附資料來源 URL，格式為「[來源](url)」或直接附 URL。若找不到可靠來源，該項必須標「⚠️ 無可靠來源，請自行確認」，並不得寫出未經查證的數字或名稱

## 報告儲存

每份 agent 報告存到 `.claude/reports/` 目錄：
- 前置調查：`agent-0-gate.md`（一律有）
- 標準方案：`agent-A.md`、`agent-B.md`
- 深度方案：`agent-R1.md` ~ `agent-R6.md`

建立或更新 `.claude/reports/index.md`，列出所有報告的主題和連結。

## 研究完成後

1. **摘要關鍵發現** — 不要只說「報告已生成」，要摘出 3-5 個最重要的發現直接告訴使用者
2. **標記時效性資訊** — 需要提前預訂的、即將截止的、有限定日期的
3. **預算初步試算** — 彙整住宿+交通+飲食+門票的概算
4. **提出需要使用者決策的問題** — 用 `AskUserQuestion` 呈現，每個問題 2-4 個選項，一次最多問 4 題。問題來自研究中發現的重要分歧點（例如：去 A 還是 B、選哪個活動、要不要租車）

## 進度追蹤

用 `TaskUpdate` 更新任務狀態（任務由 `/trip-plan` 建立）：

- 開始時：把「研究目的地」設為 `in_progress`（activeForm: 「研究中，agent 跑完會自動回報」）
- 完成時：把「研究目的地」設為 `completed`，告訴使用者下一步輸入 `/trip-go`
